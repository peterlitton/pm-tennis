# v0.8.3 release notes — OPEN-row fixes, stable match ordering, Performance link

**Release date:** 2026-05-06
**Type:** Frontend display + backend ordering changes (no schema changes, no worker logic changes)
**Scope:** Five files modified, two docs added/updated
**Rollback:** Revert the touched files; no data, schema, or worker impact

---

## What changed

This release bundles four operator-driven dashboard improvements:

### Change 1 — Investment field on OPEN row

The OPEN sub-row now displays the operator's total dollar investment in the position, between the "now" price block and the dollar P&L. Bold, no label, whole dollars only.

**Before:**
```
OPEN  100 sh @ 58¢ → now 78¢   +$20.16   +34.9%   12m
```

**After:**
```
OPEN  100 sh @ 58¢ → now 22¢   $58   −$36.00   −62.1%   12m
```

Value = `position.cost_cents / 100`, rounded to the dollar.

### Change 2 — Corrected "now" price calculation (PM-D5)

The "now" price on the OPEN row is now derived from the OPPOSING player's match-row price:

```
now_for_held_player = 100 − opposing_player's_match_row_price
```

This matches the price the operator would realize on exit. Validated against the iPhone Polymarket app. The match-row per-side prices are unchanged — only the OPEN sub-row's "now" field uses the new derivation.

Worked examples:

```
Operator holds Boulter. Match-row prices: Boulter 22¢, Lys 78¢.
  Boulter's OPEN-row "now" = 100 − 78 = 22¢

Operator holds Galfi. Match-row prices: Galfi 23¢, Potapova 77¢.
  Galfi's OPEN-row "now" = 100 − 77 = 23¢
```

### Change 3 — Stable match ordering by scheduled start time

Previously, matches rendered as "live first (dict insertion order), upcoming after (start-time order)." When a new match transitioned to live, every other row shifted, making it hard for the operator to keep visual track of any given match.

Now: all matches render in a single list sorted by `start_time` ascending, regardless of status. A match holds its vertical position across the live/upcoming transition.

To enable this, `start_time` is now populated on every match (live AND upcoming), not only on upcoming. Underlying API-Tennis `event_date` + `event_time` fields are present on every frame so no data was missing — the prior code was just discarding it.

### Change 4 — Performance link in header

A "Performance" text link is added to the header, after the open-positions count, separated by a `·` divider. Routes to `/trading?tz=America/Chicago&token=…` in a new tab.

Styled as a quiet text link (tertiary text color, dotted underline) to match the visual weight of the existing upcoming/positions count — header signal stays informational, not a CTA.

## Why bundle these four

All four are operator-driven UX/correctness improvements with no shared logic but all in the same area of the codebase (header + OPEN row + match list). Bundling reduces deploy ceremony. Each can be reverted independently if needed (see Rollback section).

## Acceptance criteria

### Investment field

1. Every OPEN row shows the dollar invested between the current price block and the dollar P&L.
2. The value is bold (`font-weight: 600`), primary text color.
3. Whole dollars only — no `.00`, no cents, no decimals.
4. On mobile flex-wrap, the investment field appears between detail and P&L.
5. Profit AND loss positions both render the field.

### "Now" price rule

6. For any match where the operator holds a position, OPEN-row "now" equals `100 − opposing_match_row_price` to the cent.
7. When opposing price is null, OPEN row shows `—` for "now".
8. Match-row per-side prices are UNCHANGED.
9. P&L numbers are UNCHANGED — still server-computed from `cost_cents` / `cash_value_cents`.

### Match ordering

10. All matches render in a single list sorted by `start_time` ascending.
11. A match's vertical position does not change when its status flips from upcoming to live.
12. Matches with a missing `start_time` sort to the top (visible-but-undated) rather than disappearing.

### Performance link

13. "Performance" link visible in the header, after the open-positions count, separated by `·`.
14. Clicking opens `/trading?tz=America/Chicago&token=…` in a new browser tab.
15. Visual weight matches the surrounding header text (tertiary color, dotted underline).

### Documentation

16. Architecture handoff doc has new section 2.7 (and the existing 2.6 carried forward).
17. PM-D5 spec doc is present in `docs/`.
18. Indicator legend on the dashboard footer mentions the new investment field and the opposing-side derivation.

## Smoke test

After deploy, open https://pm-dashboard-o71w.onrender.com/ and check:

- Header shows "Performance" link after the open-positions count, separated by `·`. Click it; new tab opens to the trading-history page.
- Match rows are sorted by scheduled start time, ascending; live and upcoming intermixed in chronological order.
- Each OPEN row shows: shares @ entry → now (derived as 100 − opposing) → bold dollar investment → dollar P&L → percent P&L → time held.
- Watch a match transition from upcoming to live (or refresh the page during a transition); its vertical position should not change.
- No console errors. No layout shift.

## Rollback

The change touches five source files plus two docs. Independent rollback per change:

- **Investment field only:** revert the `<span class="position-investment">` line in `positionSubRow`, the `investDollars` variable, the `.position-investment` CSS rule, and the mobile order: directives.
- **"Now" price only:** revert the `renderRow` call site to pass `m.p1_price_cents` / `m.p2_price_cents` directly. The `deriveNowCents` helper can stay defined harmlessly.
- **Match ordering only:** revert `state.py snapshot()` to the previous live-first/upcoming-second composite. Revert `api_tennis_worker.py` line 766 to the conditional. The `Match.start_time` field happily holds extra data either way.
- **Performance link only:** revert the `<a class="header-link">` and `<span class="header-divider">` from `templates/dashboard.html`. The `.header-link` and `.header-divider` CSS rules can stay (unreferenced).

Whole-bundle rollback: redeploy the v0.8.2 tarball.
